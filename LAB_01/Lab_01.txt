2. What is a File Descriptor (FD)?

A file descriptor is simply:

An integer representing an opened file inside a process.

Example:

int fd = open("test.txt", O_RDONLY);

Suppose:

fd = 3

Why 3?

Because:

FD	Meaning
0	stdin
1	stdout
2	stderr
3+	user opened files

So kernel returns next free descriptor.

3. Internal Kernel Journey

When you do:

open("test.txt", O_RDONLY);

The process does NOT directly access disk.

Instead:

    ↓

    ↓

    ↓

    ↓

Then kernel returns:

file descriptor integer

4. What is an inode?

An inode is:

Metadata structure describing a file.

It DOES NOT store filename.

It stores:

permissions
owner
timestamps
file size
disk block locations

Example:

ls -i file.txt

Output:

348927 file.txt

That number = inode number.

5. File Open Flow Internally

When kernel receives open():

Step 1: Path Resolution

Kernel traverses:

/

      └── prince
            └── test.txt

Each directory lookup involves inode traversal.

Step 2: Create Kernel File Object

Kernel creates:

Contains:

Step 3: Add to Process FD Table

Process has:

task_struct
   └── files_struct
         └── fdtable

Kernel inserts pointer:

fd 3 → struct file*

Returns:

3

7. Full C++ Program

#include <unistd.h>
#include <fcntl.h>
#include <sys/syscall.h>

int main() {

    const char *filename = "demo.txt";

    const char *message =
        "Hello from raw system calls!\n";

    // OPEN FILE
    int fd = syscall(
        SYS_open,
        filename,
        O_CREAT | O_WRONLY | O_TRUNC,
        0644
    );

    if (fd < 0) {
        return 1;
    }

    // WRITE FILE
    syscall(
        SYS_write,
        fd,
        message,
        30
    );

    // RESET OFFSET
    syscall(
        SYS_close,
        fd
    );

    // OPEN AGAIN FOR READING
    fd = syscall(
        SYS_open,
        filename,
        O_RDONLY
    );

    if (fd < 0) {
        return 1;
    }

    char buffer[100];

    long bytesRead = syscall(
        SYS_read,
        fd,
        buffer,
        sizeof(buffer)
    );

    // WRITE TO TERMINAL
    syscall(
        SYS_write,
        1,
        buffer,
        bytesRead
    );

    syscall(
        SYS_close,
        fd
    );

    return 0;
}

8. Why This is “Raw”

Normally:

open()

is a libc wrapper.

But:

syscall(SYS_open, ...)

directly invokes kernel syscall interface.

This is near raw userspace → kernel interaction.

10. What Happens During write()

When:

write(fd, buffer, size)

occurs:

Kernel:

validates FD
gets struct file
checks permissions
copies data from userspace
updates page cache
schedules disk flush

Important:

Usually write DOES NOT instantly write to disk.

Linux uses:

Page Cache

for performance.

19. Why System Calls are Expensive

Crossing userspace ↔ kernel boundary costs:

privilege switch
context handling
validation
memory copy

Thus:

Many small syscalls = slower
