## SQLite3 Observations

| Check | Command | Observation |
| --- | --- | --- |
| Version | `sqlite3 --version` | `3.51.0` |
| File size | `ls -lh sample.db` | `580 KB` |
| Page size | `PRAGMA page_size;` | `4096 bytes` |
| Page count | `PRAGMA page_count;` | `145` |
| Approx. DB size | `4096 * 145` | `593,920 bytes` |
| mmap disabled | `PRAGMA mmap_size=0;` | `0` |
| mmap enabled | `PRAGMA mmap_size=268435456;` | `268435456` |

### SQLite Query Timing

| Mode | Command Summary | real | user | sys |
| --- | --- | --- | --- | --- |
| Without mmap | `PRAGMA mmap_size=0; SELECT * FROM users;` | `0.01s` | `0.00s` | `0.00s` |
| With mmap | `PRAGMA mmap_size=268435456; SELECT * FROM users;` | `0.00s` | `0.00s` | `0.00s` |

Observation: mmap did not show a major difference because the test database is small and the query completes almost instantly.

## PostgreSQL Checks

PostgreSQL commands are included in `setup_postgres.sh`. The local machine did not have `psql` available on PATH during this run.

| Check | Command |
| --- | --- |
| Version | `psql --version` |
| Create DB | `createdb lab2db` |
| Page/block size | `SHOW block_size;` |
| Page count | `SELECT relname, relpages FROM pg_class WHERE relname = 'users';` |
| Table size | `SELECT pg_size_pretty(pg_relation_size('users'));` |
| Query timing | `\timing on` then `SELECT * FROM users;` |

Expected PostgreSQL default block size is usually `8192 bytes`.

## Comparison

| Feature | SQLite3 | PostgreSQL |
| --- | --- | --- |
| Architecture | Embedded database | Client-server database |
| Server process | Not required | Required |
| Storage | Single `.db` file | Managed data directory |
| Page size | `4096 bytes` observed | Usually `8192 bytes` |
| Page count | `145` pages observed | Available through `pg_class.relpages` |
| Query timing | Very fast for small local DB | Depends on server, cache, and table size |
| mmap support | Controlled using `PRAGMA mmap_size` | No SQLite-style mmap PRAGMA |
| Best suited for | Small apps, local storage, embedded use | Multi-user apps, concurrent workloads |

## Conclusion

SQLite3 is lightweight and easy to inspect because the database is a single file. PostgreSQL is more powerful for concurrent and server-side applications. In this small test, SQLite mmap had almost no visible impact because the dataset was small and already cached efficiently.
