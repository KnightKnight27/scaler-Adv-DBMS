# LAB-4: Inspecting SQLite Storage with a Campus Dataset

## Aim

This lab demonstrates how a relational dataset is stored inside a SQLite database file and how the raw file can be inspected with a hex dump. The experiment uses a small campus registration system so that table rows, index entries, and readable text are all easy to trace in the generated database file.

## Files Included

- `campus.sql`: Original SQL script that creates the schema, inserts sample records, adds indexes, and runs verification queries.
- `campus.db`: SQLite database generated by executing `campus.sql`.
- `campus.hex`: Hex dump of `campus.db`, produced from the final database file.

Both `campus.db` and `campus.hex` were generated from `campus.sql`.

## Schema Overview

The dataset models a campus with four connected tables:

- `divisions`: Stores academic divisions such as Computing Systems or Design and Media.
- `learners`: Stores student records and links each learner to one division.
- `modules`: Stores course-style modules and records which division offers each module.
- `registrations`: Connects learners to modules for a term and stores the grade or in-progress status.

Relationships used in the script:

- `learners.division_id -> divisions.division_id`
- `modules.division_id -> divisions.division_id`
- `registrations.learner_id -> learners.learner_id`
- `registrations.module_code -> modules.module_code`

The script also creates indexes on module ownership and learner-term registrations so the database file contains both table pages and index pages.

## Commands Used

Create the SQLite database from the SQL script:

```bash
sqlite3 campus.db ".read campus.sql"
```

Generate the hex dump from the database file:

```bash
xxd -g 1 -c 16 campus.db > campus.hex
```

## How to Verify the Database

Open the database and list the objects:

```bash
sqlite3 campus.db ".tables"
sqlite3 campus.db "SELECT name FROM sqlite_master WHERE type IN ('table','index') ORDER BY type, name;"
```

Check that learner rows were inserted:

```bash
sqlite3 campus.db "SELECT COUNT(*) FROM learners;"
```

Preview the SQL verification queries again if needed:

```bash
sqlite3 campus.db "SELECT learner_id, full_name, semester FROM learners ORDER BY learner_id;"
sqlite3 campus.db "SELECT registration_id, learner_id, module_code, grade FROM registrations ORDER BY registration_id;"
```

## What the Hex Dump Shows

When `campus.hex` is opened, a few storage-level details can be observed:

- The file begins with the SQLite header text, which identifies the file as a SQLite database.
- The early bytes include the page-size information used by SQLite to organize storage into fixed-size pages.
- Table rows and index records are stored across pages, so inserted data and index metadata contribute to the byte layout of the file.
- Human-readable strings such as division names, learner names, module codes, and grade values can often be spotted directly in the hex dump alongside their ASCII view.

This makes the lab useful both as a SQL exercise and as a simple introduction to how SQLite persists relational data on disk.
