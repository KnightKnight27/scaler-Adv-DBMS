# Lab 1: File Copy Using Low-Level POSIX System Calls

## 1. Objective

The purpose of this lab is to understand how file input/output works at the operating-system level. Instead of using high-level C++ file handling libraries such as `<fstream>` or standard I/O functions such as `printf`, this implementation copies data from one file to another using only the basic POSIX system calls: `open()`, `read()`, `write()`, and `close()`.

This exercise helps explain how user programs interact with the kernel through file descriptors, how data moves between disk and memory, and why system calls are the foundation behind file handling in higher-level programming languages.

## 3. Program Logic

The program follows these steps:

1. Open `input.txt` in read-only mode using `open()`.
2. Create or overwrite `output.txt` using `open()` with write permissions.
3. Read chunks of data from the input file using `read()`.
4. Write each chunk into the output file using `write()`.
5. Repeat the process until `read()` returns `0`, which means end of file.
6. Close both file descriptors using `close()`.
7. Print the total number of bytes copied and the number of `read()` calls.

## 4. Source Code

```cpp
// Lab 1: File copy using only POSIX system calls
// Allowed calls: open, read, write, close

#include <fcntl.h>
#include <unistd.h>
#include <sys/stat.h>
#include <errno.h>
#include <string.h>

static const char *INPUT_FILE = "input.txt";
static const char *OUTPUT_FILE = "output.txt";
static const size_t BUFFER_SIZE = 4096;

static int write_all(int fd, const char *buffer, size_t size) {
    size_t written_total = 0;

    while (written_total < size) {
        ssize_t written = write(fd, buffer + written_total, size - written_total);

        if (written < 0) {
            if (errno == EINTR) {
                continue;
            }
            return -1;
        }

        written_total += (size_t)written;
    }

    return 0;
}

static void print_text(int fd, const char *text) {
    write_all(fd, text, strlen(text));
}

static char *number_to_text(unsigned long long value, char *end) {
    *--end = '\0';

    if (value == 0) {
        *--end = '0';
        return end;
    }

    while (value > 0) {
        *--end = '0' + (value % 10);
        value /= 10;
    }

    return end;
}

int main() {
    int input_fd = open(INPUT_FILE, O_RDONLY);

    if (input_fd < 0) {
        print_text(STDERR_FILENO, "Error: unable to open input.txt\n");
        return 1;
    }

    int output_fd = open(OUTPUT_FILE, O_WRONLY | O_CREAT | O_TRUNC, 0644);

    if (output_fd < 0) {
        print_text(STDERR_FILENO, "Error: unable to create output.txt\n");
        close(input_fd);
        return 1;
    }

    char buffer[BUFFER_SIZE];
    unsigned long long total_bytes = 0;
    unsigned long long read_count = 0;

    ssize_t bytes_read;

    while ((bytes_read = read(input_fd, buffer, sizeof(buffer))) > 0) {
        if (write_all(output_fd, buffer, (size_t)bytes_read) < 0) {
            print_text(STDERR_FILENO, "Error: write failed\n");
            close(input_fd);
            close(output_fd);
            return 1;
        }

        total_bytes += (unsigned long long)bytes_read;
        read_count++;
    }

    if (bytes_read < 0) {
        print_text(STDERR_FILENO, "Error: read failed\n");
        close(input_fd);
        close(output_fd);
        return 1;
    }

    close(input_fd);
    close(output_fd);

    char bytes_buffer[32];
    char reads_buffer[32];

    char *bytes_text = number_to_text(total_bytes, bytes_buffer + sizeof(bytes_buffer));
    char *reads_text = number_to_text(read_count, reads_buffer + sizeof(reads_buffer));

    print_text(STDOUT_FILENO, "Copied ");
    print_text(STDOUT_FILENO, bytes_text);
    print_text(STDOUT_FILENO, " bytes using ");
    print_text(STDOUT_FILENO, reads_text);
    print_text(STDOUT_FILENO, " read() calls\n");

    return 0;
}
```

## 6. Output Verification

To verify that the copied file is identical to the original file, the byte count and checksum can be compared.

### Linux/macOS

```bash
wc -c input.txt output.txt
md5 input.txt output.txt
```

### Windows PowerShell

```powershell
Get-Item input.txt, output.txt | Select-Object Name, Length
Get-FileHash input.txt -Algorithm MD5
Get-FileHash output.txt -Algorithm MD5
```

If both files have the same size and the same hash value, the copy operation is successful.

## 7. Important Concepts Learned

### File Descriptor

A file descriptor is a small integer returned by the operating system when a file is opened. The process uses this integer to refer to the file in later system calls. Standard descriptors are `0` for input, `1` for output, and `2` for error output. Newly opened files usually receive descriptors starting from `3`.

### `read()` System Call

The `read()` system call transfers bytes from a file descriptor into a user-provided buffer. A return value greater than zero means data was read successfully. A return value of `0` means end of file, and a negative value indicates an error.

### `write()` System Call

The `write()` system call sends bytes from a memory buffer to a file descriptor. It may sometimes write fewer bytes than requested, so the program includes a loop to ensure the full buffer is written.

### Page Cache

The page cache is a memory area used by the operating system to temporarily store file data. When the same file is read again, the data may come from RAM instead of the disk, making repeated reads faster.

### Inode

An inode stores metadata about a file, such as permissions, owner, size, timestamps, and pointers to data blocks. The filename itself is stored in a directory entry, which maps the filename to an inode.

### DMA

Direct Memory Access allows hardware devices such as storage controllers to transfer data directly to RAM without requiring the CPU to manually copy every byte. This improves I/O efficiency.
